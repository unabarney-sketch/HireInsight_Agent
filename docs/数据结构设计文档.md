# 数据结构定义文档（v1.0）

> **文档目的**：为 HireInsight-Agent 数据采集与清洗模块提供严格的数据契约，锁定各数据源的原始字段与清洗后的标准格式，防止后续编码阶段产生逻辑幻觉。

---

## 一、多源原始 API JSON 结构定义

### 1.1 字节跳动招聘官网（jobs.bytedance.com）

#### 1.1.1 列表接口

**请求地址**：
```
POST https://jobs.bytedance.com/api/v1/search/job/posts
```

**请求头**（必须）：
```json
{
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    "Origin": "https://jobs.bytedance.com",
    "Referer": "https://jobs.bytedance.com/experienced/position",
    "Cookie": "atsx-csrf-token=<token>"  // 需先从 /api/v1/csrf/token 获取
}
```

**请求体**：
```json
{
    "job_category_id_list": [],
    "keyword": "",
    "limit": 10,
    "location_code_list": [],
    "offset": 0,
    "portal_entrance": 1,
    "portal_type": 2,
    "recruitment_id_list": [],
    "subject_id_list": []
}
```

## 二、标准 DataFrame 结构定义

### 2.1 目标表：`job_positions`

经 `data_transformer.py` 转换、`data_cleaner.py` 清洗后的标准宽表结构。

#### 2.1.1 字段定义表

| # | 列名 | 数据类型 | 约束 | 生产环境说明 | 来源映射 |
|---|------|---------|------|------------|---------|
| 1 | `id` | VARCHAR(32) | **PK**, NOT NULL | 雪花算法生成的唯一ID | 组合：`{source}_{original_id}` |
| 2 | `source` | VARCHAR(16) | NOT NULL, ENUM | 数据来源标识 | 固定值：`bytedance` / `tencent` |
| 3 | `original_id` | VARCHAR(64) | NOT NULL | 各平台原始岗位ID | `job_id` / `postId` |
| 4 | `title` | VARCHAR(128) | NOT NULL | 标准化职位名称 | `title` / `RecruitPostName` |
| 5 | `title_raw` | VARCHAR(256) | NULL | 原始职位名称（保留） | 用于追溯 |
| 6 | `company` | VARCHAR(64) | NOT NULL | 公司名称 | 固定值：`字节跳动` / `腾讯` |
| 7 | `department` | VARCHAR(128) | NULL | 部门/事业群名称 | `department.name` / `BGName` |
| 8 | `category` | VARCHAR(32) | NOT NULL | 职位大类 | `job_category.parent.name` / `CategoryName` |
| 9 | `sub_category` | VARCHAR(32) | NULL | 职位细类 | `job_category.name` / 空 |
| 10 | `city` | VARCHAR(32) | NOT NULL | 城市（标准化） | 清洗后：`normalize_city()` |
| 11 | `city_raw` | VARCHAR(64) | NULL | 原始城市字段 | 用于追溯 |
| 12 | `district` | VARCHAR(32) | NULL | 行政区/区域 | `city_info.district` |
| 13 | `salary_raw` | VARCHAR(64) | NULL | **原始薪资串** | 保留用于审计：`show_salary` / `Salary` |
| 14 | `salary_min` | INT | NULL, >=0 | **最低月薪（千元）** | 统计用：`parse_salary()` 返回 |
| 15 | `salary_max` | INT | NULL, >=0 | **最高月薪（千元）** | 统计用：`parse_salary()` 返回 |
| 16 | `salary_bonus` | VARCHAR(16) | NULL | 绩效倍数 | 从 `25k-40k·14薪` 提取 `14薪` |
| 17 | `experience_min` | INT | NULL, >=0 | **最小经验年限** | 统计用：`parse_experience()` 返回 |
| 18 | `experience_max` | INT | NULL, >=0 | **最大经验年限** | 统计用：`parse_experience()` 返回 |
| 19 | `experience_raw` | VARCHAR(32) | NULL | 原始经验要求 | 用于追溯：`experience.name` / `RequireWorkYearsName` |
| 20 | `degree` | VARCHAR(16) | NULL | **学历要求（标准化）** | 统计用：`normalize_education()` |
| 21 | `degree_raw` | VARCHAR(32) | NULL | 原始学历字段 | 用于追溯：`degree.name` / `DegreeName` |
| 22 | `work_type` | VARCHAR(16) | NULL | 工作类型 | 固定枚举：`全职`/`实习`/`外包` |
| 23 | `duty` | TEXT | NULL | 岗位职责（清洗后） | 存入 RAG |
| 24 | `requirement` | TEXT | NULL | 任职要求（清洗后） | 存入 RAG |
| 25 | `skills` | TEXT | NULL | 技能标签（逗号拼接） | `tags` 数组拼接 |
| 26 | `post_url` | VARCHAR(512) | NOT NULL | 原始详情页URL | 链接 |
| 27 | `published_at` | DATETIME | NULL | 发布时间（北京时间） | 解析时间戳或日期字符串 |
| 28 | `updated_at` | DATETIME | NULL | 更新时间（北京时间） | `LastUpdateTime` |
| 29 | `created_at` | DATETIME | NOT NULL | 入库时间 | 系统自动生成 |
| 30 | `crawl_batch` | VARCHAR(32) | NOT NULL | 采集批次号 | 格式：`YYYYMMDD_HHMMSS` |

#### 2.1.2 统计专用拆分字段（统计看板维度）

> 以下字段专门服务于"数据感知与大屏"的四个统计维度。

| 统计维度 | 字段 | 说明 |
|---------|------|------|
| **工作经验** | `experience_min`, `experience_max` | 支持区间分组：`0-1年`、`1-3年`、`3-5年`、`5-10年`、`10年+` |
| **地区分布** | `city` | 直接按城市分组统计 |
| **薪资区间** | `salary_min`, `salary_max` | 计算中位数后分段：`5k以下`、`5-10k`、`10-20k`、`20-30k`、`30-50k`、`50k以上` |
| **学历水平** | `degree` | 标准化枚举：`中专及以下`、`高中`、`大专`、`本科`、`硕士`、`博士` |

---

### 2.2 数据质量约束（清洗规则）

| 约束类型 | 字段 | 清洗规则 |
|---------|------|---------|
| **非空检查** | `title`, `company`, `category`, `city`, `post_url` | 任一字段为空则标记为脏数据，不入库 |
| **薪资解析** | `salary_raw` | 正则匹配格式：`xxk-xxk`、`xxK-xxK`、`xxK~xxK`、`xxk~xxk`、`面议` |
| **薪资换算** | `salary_min/max` | 统一换算为**千元/月**（年薪 ÷ 12 ÷ 1000，日薪 × 22） |
| **城市规范化** | `city_raw` → `city` | 映射表：城市名 → 一线城市/新一线/二线城市 |
| **经验解析** | `experience_raw` | 正则匹配：`经验不限`、`在校生/应届生`、`1年以下`、`1-3年`、`3-5年`、`5-10年`、`10年以上` |
| **学历标准化** | `degree_raw` → `degree` | 枚举映射：`大专及以上` → `大专`，`本科及以上` → `本科` |
| **文本清洗** | `duty`, `requirement` | 去除 `\n`、多余空格、HTML 标签 |
| **去重检查** | `original_id + source` | 唯一性约束，重复则更新而非插入 |

## 三、附录：数据字典

### 3.1 薪资原始格式与清洗示例

| 原始薪资串 | 解析后 (salary_min, salary_max) | bonus | 换算说明 |
|-----------|-------------------------------|-------|---------|
| `25k-45k` | (25, 45) | NULL | 直接取数值 |
| `30K-50K` | (30, 50) | NULL | 大写K转小写 |
| `25K~45K` | (25, 45) | NULL | `~` 替换为 `-` |
| `25000-45000` | (25, 45) | NULL | 原始分为单位 |
| `25k-40k·14薪` | (25, 40) | `14薪` | 提取绩效倍数 |
| `20-35k` | (20, 35) | NULL | 无空格格式 |
| `面议` | NULL | NULL | 无法解析 |
| `8k-12k·16薪` | (8, 12) | `16薪` | 年薪制 |
| `200-300元/天` | (5, 7) | NULL | 日薪转月薪（×22） |
