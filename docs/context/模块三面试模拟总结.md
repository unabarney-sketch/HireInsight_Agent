## 模块三：AI 面试官模拟系统

### 一句话定位

基于 LangGraph + DeepSeek + ChromaDB 的三 Agent 串行工作流，输入简历+目标岗位，输出市场分析报告 + 技能差距诊断 + 定制面试题。

---

### 架构拓扑

```
[START] → [Market_Agent] → [Critic_Agent] → [Interviewer_Agent] → [END]
```

- **图类型**：直线 DAG（`StateGraph(InterviewState)`），无条件边、无循环
- **执行模式**：同步阻塞，`app.invoke()` 串行推进三个节点
- **超时保护**：LLM 客户端层 `ChatOpenAI(timeout=120, max_retries=0)`
- **逃生通道**：图执行异常 → 自动降级为 Python 顺序链式调用 `run_interview_workflow_fallback()`

---

### 三个 Agent 节点

| 节点 | 文件 | 输入 | 输出 | 核心逻辑 |
|------|------|------|------|---------|
| **Market_Agent** | `nodes.py:86` | `target_position`, `market_summary`/`market_data` | `market_report` (Markdown) | 基于岗位市场数据生成薪资/城市/学历分析报告 |
| **Critic_Agent** | `nodes.py:191` | `user_resume`, `market_report`, `target_position` | `gap_analysis` (Markdown 表格), `rag_context` | RAG 检索 Top-2 面经 + LLM 诊断 5 维度技能差距 |
| **Interviewer_Agent** | `nodes.py:271` | `gap_analysis`, `rag_context`, `target_position` | `interview_questions` (List), `market_report` | 根据技能盲区动态生成 3-5 道场景化变形题 |

---

### State 定义（`graphs/state.py`）

```python
class InterviewState(TypedDict, total=False):
    user_resume: str               # PDF 解析后的简历文本
    target_position: str           # 目标岗位
    target_company: Optional[str]  # 目标公司
    market_data: Optional[DataFrame]
    market_summary: Optional[Dict]
    market_report: Optional[str]   # Market_Agent 输出
    gap_analysis: Optional[str]    # Critic_Agent 输出
    interview_questions: Optional[List[str]]  # Interviewer_Agent 输出
    rag_context: Optional[List[str]]          # ChromaDB 检索片段
    current_step: str              # 执行阶段追踪
    execution_error: Optional[str] # 非空时触发降级渲染
    is_completed: bool
```

---

### 关键设计决策

| 决策 | 原因 |
|------|------|
| **`ChatOpenAI` 而非 `ChatDeepSeek`** | 老版 `ChatDeepSeek` 不支持 DeepSeek API 特有参数 |
| **`deepseek-reasoner` 而非 `chat`** | 需要 Reasoning 思考链能力，`chat` 不具备 |
| **`max_retries=0`** | 不做自动重试，异常快速冒泡到 fallback |
| **`recursion_limit=10`** | LangGraph 全局步数预算，防止无限循环 |
| **双路径类型一致性** | 正常路径和 fallback 路径返回完全相同的 `InterviewState`，前端无需区分 |
| **Interviewer_Agent 复用 `market_report` 字段** | 面试题输出写入 `market_report`，前端统一从该字段渲染静态看板 |

---

### LLM 调用方式

```
get_llm_client()                     # 模块级单例，首次调用时初始化
  └─ ChatOpenAI(
       model="deepseek-reasoner",
       openai_api_base="https://api.deepseek.com/v1",
       timeout=120, max_retries=0
     )

call_deepseek(system_prompt, user_prompt)
  └─ SystemMessage + HumanMessage → llm.invoke() → response.content
```

API Key 通过 `.env` 的 `DEEPSEEK_API_KEY` 注入，`get_llm_client()` 中 `os.getenv()` 读取。

---

### RAG 集成

- **向量库**：ChromaDB `PersistentClient`，嵌入式模式（非 Docker）
- **检索函数**：`utils/rag_loader.py` → `query_experiences(query, n_results=2)`
- **降级策略**：`import` 失败或检索异常均静默跳过，`rag_context` 为空不影响 LLM 调用
- **Critic_Agent** 将检索到的面经片段注入 user_prompt，Interviewer_Agent 复用 `rag_context` 作参考

---

### 前端串联（`app.py` → `render_interview()`）

1. 用户输入岗位 + 上传 PDF → 点击"开始面试模拟"
2. `st.status` 容器展开，三个节点依次上报进度（`st.write`）
3. 执行完毕 → 静态 Markdown 看板渲染：市场报告 → 差距诊断表格 → 面试题列表
4. 异常时 `execution_error` 非空 → 前端渲染错误提示，不白屏
