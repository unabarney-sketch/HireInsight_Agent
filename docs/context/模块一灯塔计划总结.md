## 模块一：灯塔计划（低年级职业规划情景测评系统）

### 一句话定位

基于 LangGraph 两节点直线 DAG，输入目标岗位+年级 → LLM 从 10 道硬核情景多选题中动态筛选 5-7 题 → 用户作答 → AI 评估六大技术倾向（0-100 雷达图）+ 生成个性化 JSON/Markdown 双格式学习路线图。

---

### 架构拓扑

```
[用户输入岗位+年级] → [QuestionFilterNode] → [前端渲染 5-7 题] → [用户答题]
                                                                    ↓
[Plotly 雷达图 + Markdown 路线图] ← [AssessmentNode] ← [st.form 提交]
```

- **图类型**：直线 DAG（`StateGraph(LighthousePlanState)`），两节点串联，无条件边、无循环
- **执行模式**：同步阻塞，Phase 1 直接调用 `question_filter_node()`，Phase 2 直接调用 `assessment_node()`（不经过 LangGraph `invoke`，而是节点级直接调用，降级时走纯 Python fallback）
- **超时保护**：LLM 客户端层复用模块三的 `ChatOpenAI(timeout=120, max_retries=0)`

---

### 两个 Agent 节点

| 节点 | 文件 | 输入 | 输出 | 核心逻辑 |
|------|------|------|------|------|
| **QuestionFilterNode** | `lighthouse_nodes.py:75` | `target_position`, `grade`, `all_questions` | `filtered_questions` (5-7 题) | LLM 根据岗位语义和题目 tags 筛选，LLM 失败降级为 10 题全量 |
| **AssessmentNode** | `lighthouse_nodes.py:192` | `user_answers`, `target_position`, `grade` | `tech_tendency`, `roadmap_json`, `roadmap` (MD) | 3 层 JSON Fallback 解析 + try-except 超时降级 |

### State 定义（`graphs/state.py:44`）

```python
class LighthousePlanState(TypedDict, total=False):
    # === 用户输入 ===
    target_position: str                    # 目标岗位
    grade: str                              # 当前年级

    # === 题目数据 ===
    all_questions: List[dict]               # 完整题库 10 题
    filtered_questions: List[dict]          # LLM 筛选后 5-7 题

    # === 用户答题 ===
    user_answers: List[dict]                # [{question_id, selected_option, tendency}]
    user_choices: List[int]                  # 保留兼容

    # === Agent 输出 ===
    tech_tendency: Optional[Dict[str, float]]   # 六维度倾向（0-100）
    roadmap_json: Optional[Dict[str, Any]]      # JSON 结构化路线图
    roadmap: Optional[str]                      # Markdown 路线图

    # === 执行状态 ===
    current_step: str
    execution_error: Optional[str]          # 非空+is_completed=True → 前端降级渲染
    is_completed: bool
```

**关键约定**：
- `execution_error` + `is_completed=True` → 前端跳过雷达图，展示降级错误页
- JSON 解析失败不算 `execution_error`（降级为纯文本 `roadmap`），只有 LLM 超时/网络异常才写 `execution_error`
- ⚠️ **禁止混用 `@dataclass` 和 `TypedDict`**：原代码有 `@dataclass` 装饰器在 `LighthousePlanState` 和 `MarketDataState` 上，已修复为纯 `TypedDict`

### 两个必须防范的技术陷阱（已全部落地）

#### 陷阱 1：`st.radio` 重运行导致 LLM 重复调用

- **根因**：Streamlit 每次 radio 点击都全脚本重运行
- **措施 1**：Phase 2 所有 `st.radio` 包裹在 `st.form("lighthouse_quiz_form")` 中，只有 submit 触发执行
- **措施 2**：`filtered_questions` 在 QuestionFilterNode 执行后立即存入 `st.session_state.lighthouse_filtered_questions`，re-run 时优先读缓存，零 API 调用
- **措施 3**：`user_answers` 同样走 session_state 中转

#### 陷阱 2：DeepSeek Reasoner 思考链（CoT）污染 JSON 输出

- **根因**：`deepseek-reasoner` 先输出 CoT 推理过程，再输出 JSON 结果
- **多层 Fallback**（`lighthouse_nodes.py:11 _extract_json_from_reasoner_response()`）：
  - 第 1 层：正则 `r'```json\s*(\{.*?\})\s*```'`（re.DOTALL）提取 fenced code block
  - 第 2 层：`find('{')` + `rfind('}')` 首尾截取
  - 第 3 层：全量失败 → 原始文本写 `roadmap`，`roadmap_json = None`，**不重试 LLM**

#### 陷阱 3：超时逃生通道

- 两节点内 `try-except` 包裹 LLM 调用，异常时返回 `execution_error` + `is_completed=True`
- 节点内 `st.write()` 包裹 `if not state.get("execution_error"):` 防 fallback 重放重复日志

### 题库设计（`utils/question_bank.py`）

- **10 道硬核情景多选题**，覆盖：性能优化(2) / 架构选型(2) / Bug排查(1) / 技术债治理(1) / 数据处理(2) / 项目规划(1) / 安全攻防(1)
- **每道题 4 个选项**，tendency 严格限制为六者之一（`前端/后端/AI数据/测试运维/产品/客户端`），同一题内 4 个 tendency 互不相同
- **硬核工程痛点导向**（非空泛行业黑话）：OS epoll/eBPF、哈斯图偏序集、YOLO TensorRT 量化、Java 线程池 + JVM GC、False Sharing/MESI、CRDT/OT 冲突解决 等
- 辅助函数：`get_full_question_bank()`, `format_questions_for_llm()`, `get_question_by_id()`, `get_questions_by_category()`, `get_valid_tendencies()`

### 前端串联（`app.py` → `render_lighthouse()`）

```
Phase 1-Step 1: 用户输入 target_position + grade
                 → 点击"开始测评"
                 → st.status 包裹 question_filter_node()
                 → filtered_questions 写入 st.session_state

Phase 1-Step 2: st.form 渲染 5-7 道 st.radio 单选题
                 → 点击"生成路线图"
                 → 校验全部作答
                 → st.status 包裹 assessment_node()
                 → 结果写入 st.session_state.lighthouse_results
                 → st.rerun()

Phase 2:        检测 lighthouse_results 存在
                 ├─ execution_error + 无 tech_tendency + 无 roadmap → st.error() 降级页
                 ├─ execution_error + 有 roadmap（纯文本） → st.warning() + st.markdown()
                 └─ 正常 → Plotly 雷达图 (go.Scatterpolar, 蓝色半透明) + Markdown 路线图
```

**session_state 缓存变量**（跨 re-run 和跨页面切换）：
- `lighthouse_filtered_questions` — 选题结果缓存
- `lighthouse_results` — 评估结果（None = 未完成）
- `lighthouse_quiz_started` — 阶段切换标志
- `lighthouse_target_position` / `lighthouse_grade` — 用户输入持久化
