#### 1. 修复 mock 数据 `original_id` NULL 约束报错

**文件**：`app.py` 第 610 行

**问题**：`save_to_sqlite` 入库时报 `IntegrityError: NOT NULL constraint failed: job_positions.original_id`。

**根因**：mock 数据字典（第 608-627 行）遗漏 `original_id` 字段，`clean_job_data` Step 8 的条件生成逻辑（`if "source" in df.columns and "original_id" in df.columns`）在字段缺失时静默跳过，不会自动补全。

**修复**：在 `"id"` 后新增一行 `"original_id": f"mock_job_{i}_{j}"`。

```python
# app.py 第 608-610 行（修改后）
raw_jobs.append({
    "id": f"mock_job_{i}_{j}",
    "original_id": f"mock_job_{i}_{j}",   # ← 新增
    "title_raw": f"Python后端开发工程师-{city}",
    ...
})
```

**连锁效果**：`clean_job_data` Step 8 检测到 `original_id` 存在后，会将 `id` 重新生成为 `{source}_{original_id}`（如 `mock_bytedance_mock_job_0_0`），对齐 `(original_id, source)` UNIQUE 索引语义。

**原则记录**：新增 SQLite DDL 中 NOT NULL 列时，**必须同步更新 mock 数据字典的所有条目**，否则运行时必炸。
