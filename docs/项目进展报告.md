# HireInsight-Agent 项目当前进展情况

## 项目概述

HireInsight-Agent 是一个面向校招/社招场景的智能求职辅助系统，基于 Streamlit + LangGraph + DeepSeek API + RAG（TF-IDF）技术栈构建。核心理念是"用 AI 帮学生更好地看清市场、看清差距、看清自己"。目前项目已进入功能完备阶段，三大核心模块全部实现并可运行。

## 核心模块完成情况

### 1. 数据大屏模块（已完成 100%）
- 多维度筛选器：支持城市、学历、公司来源（腾讯/网易/字节/滴滴/美团）、薪资范围、经验要求、关键词搜索等多条件组合筛选
- Plotly 可视化：薪资分布柱状图、城市岗位横向条形图、学历/经验饼图，数据动态渲染
- 数据管线：爬虫抓取 → Transformer 策略路由 → Cleaner 8 步清洗 → SQLite Upsert 入库 → 可视化渲染，全链路打通
- 缓存优化：st.cache_data 装饰器实现 5 分钟 TTL 缓存，避免重复查询 SQLite

### 2. 灯塔计划模块（已完成 100%）
- 两阶段交互：用户输入目标岗位与年级，AI 从 10 道情景多选题中动态筛选 5-7 题；用户答题后 AI 评估六维度技术倾向（前端/后端/AI数据/测试运维/产品/客户端，0-100 分）
- 可视化：Plotly 雷达图 + 结构化 Markdown 学习路线图
- LangGraph 工作流：QuestionFilterNode → AssessmentNode 直线 DAG 拓扑
- 历史记录：自动保存测评结果到 SQLite，支持查看和删除

### 3. 面试模拟模块（已完成 100%）
- 三 Agent 串联：Market_Agent（市场趋势分析）→ Critic_Agent（简历差距诊断 + RAG 检索）→ Interviewer_Agent（动态生成定制化面试题）
- PDF 简历解析：pdfplumber 提取文本内容
- RAG 增强：TF-IDF + 余弦相似度检索 Top-2 企业面经片段，为差距诊断提供上下文
- 容错降级：LangGraph 图执行异常时自动 fallback 为 Python 顺序链式调用
- 追问沙盒：结果展示后可进行 AI 追问对话

### 4. 用户认证与个人中心（已完成 100%）
- 完整的注册/登录/注销体系：PBKDF2-SHA256 哈希密码（10 万次迭代 + 随机 Salt），SQL 参数化防注入
- 商业级登录页：纯 CSS 背景画卷 + 右侧表单浮层设计
- 个人中心：三栏布局（名片 + 资产统计 + 修改表单），支持头像、显示名、年级修改
- 多用户隔离：所有历史记录查询按 user_id 隔离，保障数据安全

### 5. 系统设置模块（已完成 100%）
- DeepSeek API Key 配置与持久化（写入 .env）
- 环境变量热重载，无需重启应用

## 当前存在的问题

### 1. 数据时效性不足
- 爬虫数据需要手动触发抓取，缺乏定时自动更新机制，数据可能随时间推移逐渐过时
- 部分招聘网站页面结构变化频繁，爬虫存在随时失效的风险，缺少爬虫健康监控与告警机制
- 不同数据源抓取成功率不均衡，部分 Playwright 爬虫可能因反爬策略升级导致抓取失败

### 2. 题库与面经库覆盖度有限
- 灯塔计划题库仅包含 10 道情景多选题，题目数量和覆盖维度有待扩充，长期使用可能产生重复感
- TF-IDF 检索依赖本地企业面经文件数量，当前面经库数据量较小，可能影响 Critic_Agent 差距诊断的深度和准确性
- 缺少用户对题目质量和诊断结果的反馈收集机制，无法形成"使用 → 反馈 → 优化"的闭环

### 3. RAG 检索方案存在局限
- TF-IDF + 余弦相似度虽解决了 ChromaDB 兼容性问题，但作为稀疏向量方案，语义理解能力弱于密集向量模型，对同义词和上下文理解有限
- 无法进行多模态检索（如对简历中的项目经历关键词与面经进行交叉匹配）
- 面经数据未做分块优化和元数据标注，检索粒度较粗

### 4. LLM 强依赖与容错不足
- 灯塔计划和面试模拟模块深度依赖 DeepSeek API，若 API 不可用或用户未配置 Key，核心功能完全不可用
- 当前虽有针对 LLM 调用超时的 try-catch fallback，但对 API 返回格式异常（非 JSON、字段缺失）的处理仍需加强
- 国际网络环境下访问国内 API 可能存在延迟问题，缺少本地缓存策略

### 5. 用户体验与交互细节待完善
- Streamlit 每次交互都会触发全页重新渲染，大页面（如数据大屏）刷新时闪烁感明显
- 移动端响应式体验一般，部分 Plotly 图表和复杂表格在小屏幕上显示效果不佳
- 面试模拟模块中，PDF 简历上传后若文件过大（如包含大量图片的简历），解析速度和内存占用未做限制
- 缺少操作引导和帮助文档，新用户上手存在一定学习成本

### 6. 部署与运维短板
- 当前仅支持 `streamlit run` 本地启动，缺少 Docker 化部署方案和云端部署文档
- 缺少日志收集和异常监控系统，线上问题排查依赖人工复现
- 数据库为本地 SQLite，不支持多实例并发访问，扩展性受限
- 缺少 CI/CD 流水线和自动化测试覆盖，回归测试依赖手工验证

### 7. 代码工程化水平有待提升
- `app.py` 单文件超 3100 行，页面路由、业务逻辑、样式代码混杂，维护成本随功能增长持续上升
- 部分模块缺少单元测试，`tests/` 目录仅含 3 个测试文件
- 爬虫、Transformer、Cleaner 等模块的错误处理策略不统一，部分位置异常被静默吞掉
- 缺少类型注解和 docstring 规范，代码可读性在跨模块协作时略有不足
